# DSA Pattern Classifier

Use this skill when the goal is to identify the most likely algorithm patterns for a coding interview problem.

## Responsibilities

- Extract the core shape of the problem
- Map it to common DSA patterns such as DFS, BFS, DP, Graph, Heap, Sliding Window, Two Pointers, Binary Search, Greedy, Backtracking, Trie, or Union Find
- Explain why the selected patterns fit the constraints and input structure
- Avoid jumping directly into full code unless explicitly requested

## Output Style

- Be concise and interview-oriented
- Prioritize the top one to three patterns
- Explain tradeoffs clearly
- Mention edge cases that could change the best pattern choice
